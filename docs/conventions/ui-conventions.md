# UI/UX conventions

Живой свод договорённостей по интерфейсу log-viewer. Дополняется по мере появления
правил — держим консистентность UI в одном месте, а не в головах.

Когда правило крупное, спорное или отменяет очевидную альтернативу — заводим
[ADR](../adr/) и ссылаемся на него из соответствующего раздела. Мелкие конвенции
(отступы, размеры, тексты подписей) живут прямо здесь.

Оформление — как и вся документация (см. [CLAUDE.md](../../CLAUDE.md) → Documentation):
диаграммы только Mermaid, **без ASCII-арта**; примеры — таблицами и JSX-сниппетами.

## Forms

### Раскладка «лейбл ↔ поле»

Единственный санкционированный способ сверстать подписанное поле —
компонент [`<LvFormField>`](../../src/ui/components/common/LvFormField.tsx).
Прямая вёрстка на `.lv-form-row` / `.lv-tset-vf-*` в новых формах не вводится —
так раскладка остаётся одинаковой во всех окнах.

Ориентация выбирается по ширине контейнера:

| Контекст                           | Ориентация           | Лейбл                     | Класс               |
| ---------------------------------- | -------------------- | ------------------------- | ------------------- |
| Модальное окно (нормальная ширина) | `row` (по умолчанию) | слева, фикс. колонка 80px | `.lv-form-row`      |
| Узкая боковая панель               | `column`             | сверху, на всю ширину     | `.lv-form-row--col` |

Правило: **в пределах одного окна — одна ориентация**. Модалки используют `row`
(это и есть «согласованный» вид, к которому приведены `LvAddSourceModal` и
`LvLogicalFieldEditorModal`); узкие панели вроде `LvParsersPanel` — `column`,
потому что колонка лейблов в 80px зажимает инпуты и особенно `textarea`.

```tsx
<LvFormField label="Source name" htmlFor="src-name" error={taken ? 'Name in use' : null}>
  <input id="src-name" className="lv-form-input" value={name} onChange={…} />
</LvFormField>
```

- Контрол передаётся как `children` и сам несёт класс `.lv-form-input`.
- `htmlFor` + совпадающий `id` на контроле обязательны для одиночных полей —
  клик по подписи фокусирует контрол. Опускаем только для составных строк
  (тогглы, file-picker), где единственного целевого `id` нет.

### Help и ошибки

- Подсказка (`help`) и инлайновая ошибка (`error`) рендерятся компонентом
  **под** контролом, на своей строке (классы `.lv-form-help` / `.lv-form-error`).
  Не раскладываем их вручную.
- Многострочный контрол (`textarea`) в `row`-режиме: проп `alignTop` прижимает
  подпись к первой строке (`.lv-form-row--top`), иначе она центрируется по высоте.

### Составные и списочные поля (исключения)

Через `<LvFormField>` НЕ верстаются — занимают всю ширину:

- **Парные поля** — два контрола в одной строке (например `Id` + `Kind`,
  `Timestamp` + `Transform` в редакторе парсеров). Раскладка `.lv-parsers-row-pair`
  - `.lv-form-half`.
- **Списки / секции** — повторяющиеся под-строки с заголовком сверху
  (например `Extractors` в редакторе логических полей).
